## Module Responsibility

`@orgtrace/core` is the pure shared-contract layer. It owns:
- All shared TypeScript types (`ComponentRef`, `DependencyResult`, `DependencyReference`, `RiskScore`, etc.)
- `RiskEngine` — the scoring algorithm and default ruleset
- `MetadataRegistry` — the registry of known metadata type definitions

No Node.js APIs (`fs`, `path`), no `vscode`, no browser APIs. This package must be importable by the CLI, IDE extension, a browser extension, and any future adapter.

## Entry Points

- `src/index.ts` — re-exports all public types and functions
- `src/types.ts` — all shared type definitions
- `src/RiskEngine.ts` — `calculateRisk()`, `defaultRiskRules`, `RiskRule` interface
- `src/MetadataRegistry.ts` — `registerMetadataType()`, `getMetadataTypeDefinition()`, `getAllMetadataTypes()`

Built with `tsup` to `dist/` (both CJS and ESM).

## Key Interfaces

`DependencyResult` is the central contract that every surface produces and consumes. `RiskScore` embeds inside it. `ComponentRef` identifies any Salesforce metadata component. `DependencyReference` describes a directed edge with `source`, `target`, `relationshipType`, `confidence`, and `dataSource`.

---

## Risk Scoring

`calculateRisk()` takes `Omit<DependencyResult, 'risk'>` and an optional rule array. It:
1. Adds weighted scores per reference (High=3, Medium=2, Low=1) and per dependency (half weight).
2. Evaluates each `RiskRule.match()` and adds `scoreIncrease` up to a cap of 100.
3. Maps score to level: Low (<25), Medium (<50), High (<75), Critical (>=75).

To add a risk rule, append to `defaultRiskRules` in `RiskEngine.ts` — no other files need changing.

---

## MetadataRegistry

Call `registerMetadataType(def)` at module load time (bottom of `MetadataRegistry.ts`). Currently registers 18 types (ApexClass through Unknown). To add a new type, add a `registerMetadataType(...)` call here and a parser in `packages/scanner`.
